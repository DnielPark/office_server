Kimi K2.5 OCR 연동 계획

📋 변경 사항 3개
UI sunday_chat.html
1. 📷 첨부 버튼 기능 활성화 (현재 Phase 3 placeholder alert 제거) 2. 숨겨진 <input type="file" accept="image/*"> 연결 3. 업로드 중 프로그레스 표시 (선택) 4. OCR 결과를 채팅 메시지로 자동 포맷팅하여 삽입
API file_server_external.py — 신규 엔드포인트
POST /api/ocr/upload — 이미지 업로드 + OCR - Content-Type: multipart/form-data - 필드: image (파일), prompt (옵션, 기본: "이 이미지에서 한글 텍스트를 모두 추출해줘") - Kimi K2.5 API로 이미지(base64) 전송 - 응답: { "text": "OCR 결과 텍스트" }
환경 .env — Kimi API 키 추가
KIMI_API_KEY=sk-xxxxx — Moonshot AI API 키 추가 필요
🔌 API 스펙 2개
POST
/api/ocr/upload
이미지를 업로드하면 Kimi K2.5 멀티모달 API로 OCR 수행.
요청: multipart/form-data — image (필수), prompt (선택)
응답: { "success": true, "text": "OCR 결과..." }
POST
/devnote/doc-builder/sunday/chat
기존 엔드포인트 — 프론트에서 OCR 결과를 messages에 시스템 메시지로 포함시켜 전송하면 DeepSeek이 해석하여 JSON 필드 업데이트.
📚 Kimi K2.5 API 참고
# Kimi API (Moonshot AI) — OpenAI 호환 POST https://api.moonshot.cn/v1/chat/completions Headers: Authorization: Bearer sk-xxxxx Content-Type: application/json Body: { "model": "kimi-k2.5", # 또는 moonshot-v1-8k "messages": [ { "role": "user", "content": [ { "type": "image_url", "image_url": { "url": "data:image/jpeg;base64,..." } }, { "type": "text", "text": "이 문서에서 한글 텍스트를 모두 추출해줘" } ] } ], "temperature": 0.1, "max_tokens": 2000 }
